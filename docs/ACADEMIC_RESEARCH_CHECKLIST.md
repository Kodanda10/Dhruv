# ✅ Academic Research Access Application Checklist

**Quick reference checklist for applying for Twitter Academic Research API access**

---

## Pre-Application

- [ ] **Verify Eligibility**
  - [ ] Academic affiliation confirmed (university/college)
  - [ ] Role qualifies (student/faculty/researcher)
  - [ ] Research project clearly defined
  - [ ] Non-commercial use confirmed

- [ ] **Check Current Status**
  - [ ] Visit Twitter Developer Portal
  - [ ] Verify Academic Research is still available
  - [ ] Check for any pricing changes
  - [ ] Review latest terms and conditions

- [ ] **Prepare Materials**
  - [ ] Research project description written
  - [ ] Academic affiliation proof ready
  - [ ] Data usage plan prepared
  - [ ] Non-commercial use statement drafted

---

## Application Steps

- [ ] **Step 1: Twitter Developer Account**
  - [ ] Create account at https://developer.twitter.com/
  - [ ] Complete profile (public, verified)
  - [ ] Ensure profile is in good standing

- [ ] **Step 2: Basic Developer Access**
  - [ ] Create new project
  - [ ] Select use case
  - [ ] Submit basic application
  - [ ] Wait for approval (1-3 days)

- [ ] **Step 3: Academic Research Application**
  - [ ] Navigate to Products → Academic Research
  - [ ] Click "Apply" or "Get Started"
  - [ ] Fill out application form
  - [ ] Submit application

---

## Application Information Required

- [ ] **Academic Affiliation**
  - [ ] Institution name
  - [ ] Your role/position
  - [ ] Department/Program
  - [ ] Academic email address
  - [ ] Proof of affiliation (if required)

- [ ] **Research Project**
  - [ ] Research title
  - [ ] Research objectives
  - [ ] Research questions/hypotheses
  - [ ] Methodology
  - [ ] Expected outcomes
  - [ ] Timeline

- [ ] **Data Usage Plan**
  - [ ] How you'll collect data
  - [ ] How you'll analyze data
  - [ ] How you'll store data
  - [ ] How you'll share findings
  - [ ] Data retention policy

- [ ] **Non-Commercial Statement**
  - [ ] Explicit non-commercial use statement
  - [ ] No commercial applications
  - [ ] No resale/redistribution

- [ ] **IRB/Ethics (if applicable)**
  - [ ] IRB approval number
  - [ ] Ethics approval documentation

---

## After Submission

- [ ] **Monitor Application**
  - [ ] Check email regularly
  - [ ] Monitor Developer Portal status
  - [ ] Respond to any requests promptly

- [ ] **Wait for Approval**
  - [ ] Typical timeline: 2-4 weeks
  - [ ] Be patient during review
  - [ ] Follow up if no response after 4 weeks

---

## After Approval

- [ ] **Update Code**
  - [ ] Switch to Full-Archive Search endpoint
  - [ ] Update queries to use `from:username` format
  - [ ] Test with date ranges
  - [ ] Verify access to older tweets

- [ ] **Start Fetching**
  - [ ] Test with small date ranges first
  - [ ] Verify data quality
  - [ ] Begin full data collection
  - [ ] Monitor rate limits
