## Plan and Approach
### Overall Plan
The enhancement transforms the basic rule-based parsing into an LLM-powered, ML-augmented system using LangExtract (for structured extraction) and Milvus (for vector database storage/search). This addresses gaps like missing values in the dashboard PDF (e.g., blanks in "कहाँ"/"क्या") and enables intelligent features (context inference, sentiment, purpose detection).

- **Phases**:
  1. **Data Preparation**: Use the 271 posts JSON as input. Auto-label with LangExtract for training data.
  2. **Extraction with LangExtract**: Define prompts for extracting entities (e.g., schemes, events, inferred tags like "#किसान" from "किसानों के साथ") with offsets for grounding.
  3. **Storage and Search with Milvus**: Store parsed data as vectors + metadata in Milvus for hybrid queries (semantic + filtered, e.g., "महिला सशक्तिकरण" search returning posts with inferred themes).
  4. **ML Training**: Fine-tune IndicBERT on labeled data for classification (context/themes, sentiment, purpose). Use for inference on new posts.
  5. **Integration**: Flask API endpoints for parsing/querying; Next.js dashboard consumes API for dynamic updates.
  6. **Testing/Deployment**: Separate branch for engine; CI with TDD, gates (coverage, perf LCP <=2.5s, a11y), Vercel audits. Merge post-success.
- **Approach**:
  - **LangExtract Integration**: Use few-shot prompts with examples from OP tweets (e.g., "महतारी वंदन योजना" post for scheme extraction). Chunk long tweets, parallel process for efficiency.
  - **Milvus Integration**: Create collection with schema (id, timestamp, content, embedding, parsed_metadata like "themes": ["महिला सशक्तिकरण"]). Index for cosine similarity; hybrid search (e.g., filter by sentiment=positive, semantic match to "कल्याण योजना").
  - **ML Component**: Train on 80% of 271 posts (labeled via LangExtract); validate on 20%. Tasks: multi-label classification for themes/purpose, 3-class sentiment.
  - **Holistic Enhancements**: Add sentiment (positive/negative/neutral), purpose (e.g., "wish", "promote", "condolence"), context summary. Reduce blanks via inference/web enrichment.
  - **Reversibility/Safety**: Flags for LangExtract (`FLAG_LANGEXTRACT='on'`), Milvus (`FLAG_MILVUS_SEARCH='on'`); rollback by toggling to rule-based.
  - **Tools/Dependencies**: Python (re, indic-nlp-library, pandas, torch, transformers for IndicBERT, langextract, pymilvus). No new installs; assume available.
  - **Timeline**: 4 weeks (Prep: Week 1, Extract/Store: Week 2, Train/Infer: Week 3, Test/Integrate: Week 4).
  - **Risks/Rollback**: Low risk (static data); rollback: Vercel rollback (<=10 min), flag off.
